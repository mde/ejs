Features

Geddy has you covered

Models

A flexible data layer

Geddy lets you define reusable models for your data. It's got validation, instance methods, and static methods. It'll even stub out a model adapter for you to help you get your data slurped in.

var User = function () {
  this.property('login', 'string', {required: true});
  this.property('password', 'string', {required: true});

  this.validatesPresent('login');
  this.validatesConfirmed('password', 'confirmPassword');

  // You can define methods for instances like this
  this.someMethod = function () {
    // Do some stuff
  };
};

User = geddy.model.registerModel('User', User);

Views

Support for multiple template languages

Geddy supports multiple template languages such as Jade, Handlebars, Mustache and EJS. You don't have to do anything to switch between them just change the file extension and Geddy will handle what engine to use.

Geddy includes a EJS engine so you don't have to install anything to use EJS, it's ready out of the box.

// view logic
<% if (localVar) { %>
  // do some stuff with that local variable
<% } %>

// display the value of a variable
<%= user.login %>

// just use regular js
<% for (var i in user.posts) { %>
  <%= user.posts[i].title %>
<% } %>

View Helpers

Useful View Helpers

Geddy has multiple view helpers for use with HTML tags, text manipulation and url generation. Geddy also has action which are helpers that create links to various routes defined in your routes file.

Note: View helpers are pretty new, if you'd like others please suggest them, we'd gladly add any new helpers.

// HTML tags
scriptLink('/js/script.js');
// => <script src="/js/script.js"></script>

// URL generation
urlFor({ host: 'somehost.com', protocol: 'https' })
// => 'https://somehost.com'

// Text manipulation
truncate('Once upon a time in a world')
// => 'Once up...'

Controllers

Simple controllers

When you generate a resource, your controller is automatically filled with your basic CRUD actions. If you want to add more actions to your controller, just add a method onto the controller and add a new route. It's that easy.

var User = function () {
  this.index = function (req, resp, params) {
    this.respond(params, {
      format: 'html'
    , template: 'app/views/main/index'
    });
  };
};

exports.User = User;

Content Negotiation

Automatic json and jsonp support

Geddy handles content formats for you - add a .json to your url, and you'll get json back. Add a .js?callback=foo to your url and you'll have a jsonp response. Awesome, huh?

this.respondsWith = ['js', 'json'];

this.show = function (params) {
  // (Fetch some item by params.id)
  item = {foo: 'FOO', bar: 1, baz: false};
  this.respond(item);
};

Process Clustering

No code needed for clustering

Geddy automatically handles multi-process clustering for you. It works with 0.6.x and it's backwards compatible with node 0.4.x; scale your node apps across your server's core without any extra work.

  // just set the number of workers in your config
  workers: 1

  // Geddy also handles process rotation
, rotateWorkers: true
, rotationWindow: 7200000
, rotationTimeout: 300000

App Metrics

Automatically Instrumented Controllers

With geddy, all your controller actions and requests are instrumented automatically. All you need to do is enable metrics in your config. hit up port 4001 /metrics to collect them periodically.

// counters
var counter = new geddy.metrics.Counter;
counter.inc(1);

// timers
var timer = new geddy.metrics.Timer;
timer.mark();

Generators

Geddy generators are powerful and very helpful when creating applications

Geddy generators are extremely useful in Geddy, creating new applications, scaffolding and resources fast. There are numerous generators that will make it easy to create new pieces to exists applications like scaffold, resource, model, controller and application secret generators. 

// generate an app
$ geddy app test_app

// generate a new resource
$ geddy resource user name:string

// generate a new scaffolding
$ geddy scaffold user name:string password:string

// generate a new application secret
$ geddy secret

// start your app
$ geddy

Bootstrap and jQuery

Your app automatically works on mobile devices

Geddy gives you Twitter's Bootstrap on the front end so you can rapidly lay out your prototype. Plus, since responsive layouts are built right in, your app automatically works on mobile devices and tablets.

<script src="js/jquery.min.js"></script;>
<script src="js/bootstrap.min.js"></script;>

<link rel="stylesheet" href="css/bootstrap.min.css">
<link rel="stylesheet" href="css/bootstrap.responsive.css">

Logging

Access, stdout, and error logs

Geddy automatically logs requests to an access log, and you can log anything you'd to stdout or a file. It supports 9 different log levels from debug to emergency.

// info
geddy.log.info('Some info for you');

// warning
geddy.log.warn('A warning!');

// error
geddy.log.error('An error occurred');

Routes

Easy to use Routing API

Routes are easy with Geddy. Just match a url to a controller and an action. You can specify http methods, formats, and url params.

// standard main route
router.match('/').to({controller: 'Main', action: 'index'});

// a custom route
router.match('/moving/pictures/:id').to({controller: 'Moving', action: 'pictures'});

// a restful resource route
router.resource('users');
